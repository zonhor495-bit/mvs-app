## 💡 Быстрые ответы

### Q: Как работает система обновлений?
**A:** Electron Main процесс проверяет latest.yml на сервере каждый час, если версия новее - отправляет событие в React компонент, который показывает красивый диалог. Пользователь кликает "Обновить" - начинается фоновая загрузка с прогресс-баром. Когда загружено - перезапускает приложение с новой версией.

### Q: Где находится код?
**A:**
- Main процесс: `electron/main.ts` (функция setupAutoUpdater)
- Preload API: `electron/preload.ts` (window.electron.updater)
- UI компонент: `src/components/UpdateDialog.tsx` (150 строк)
- Интеграция: `src/app/InternalApp.tsx`
- Типизация: `src/electron.d.ts`

### Q: Как выпустить обновление?
**A:**
1. Измените версию в package.json ("1.0.0" → "1.0.1")
2. Запустите `npm run build`
3. Загрузите latest.yml и MVSSetup-*.exe на https://updates.mvs.app/releases/
4. ✅ Готово! Пользователи получат обновление автоматически

### Q: Где смотреть логи?
**A:** `%APPDATA%/MVS/startup.log` (Windows)
Или: `~/.config/MVS/startup.log` (Linux)
Или: `~/Library/Application Support/MVS/startup.log` (macOS)

### Q: Что если сервер недоступен?
**A:** Ошибка логируется, приложение продолжает работать. Проверка повторится через час.

### Q: Может ли пользователь отклонить обновление?
**A:** Да, нажав "Напомнить позже". Приложение продолжит работу, проверка повторится через час.

### Q: Как откатиться на предыдущую версию?
**A:** Администратор обновляет latest.yml, указывая старую версию и файлы. Пользователи получат "обновление" на старую версию.

### Q: Работает ли без интернета?
**A:** Нет, требуется интернет для проверки обновлений и загрузки файла.

### Q: TypeScript компилируется?
**A:** ✅ Да, без ошибок (npx tsc --noEmit)

### Q: Build работает?
**A:** ✅ Да, npm run build завершается успешно
