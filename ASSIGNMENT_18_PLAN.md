## 🎯 Фаза 1: Выбор и настройка сервера обновлений

### Варианты хранилища

| Вариант | Преимущества | Недостатки | Рекомендация |
|---------|-------------|-----------|-------------|
| **GitHub Releases** | Бесплатно, легко интегрировать | Зависит от GitHub, публичный | ✅ Отлично для OSS |

## 📐 Фаза 2: Подготовка к первой публикации

### Структура файлов для обновлений

```
updates.mvs.app/releases/
├── latest.yml                 # Метаинформация текущей версии
├── MVSSetup-1.0.0.exe        # Текущая версия
├── MVSSetup-1.0.0.exe.blockmap
├── MVSSetup-1.0.1.exe        # Новая версия
└── MVSSetup-1.0.1.exe.blockmap
```

### Требования для первой публикации

1. **latest.yml** — Должен содержать:
   - Актуальную версию приложения
   - Правильный SHA512 хеш
   - Размер файла
   - Дату релиза

2. **Установщик (exe)** — Должен:
   - Быть скомпилирован с правильной версией
   - Иметь валидный цифровой подпись (опционально)
   - Быть доступным по HTTPS

3. **Blockmap** — Служит для:
   - Дельта-обновлений
   - Ускорения загрузки (скачивается только разница)

## 🔄 Фаза 3: Процесс тестирования 1.0.0 → 1.0.1

### Тестовый сценарий

```
1. ✅ Установлена версия 1.0.0
   └─ Проверить: app.getVersion() = "1.0.0"

2. ✅ Собрана и опубликована версия 1.0.1
   └─ Проверить: latest.yml содержит version: 1.0.1

3. ✅ Приложение проверило обновления
   └─ Проверить: В логе "Update available: 1.0.0 → 1.0.1"

4. ✅ Диалог обновления показан
   └─ Проверить: UpdateDialog отображает версии

5. ✅ Загрузка началась
   └─ Проверить: Download progress 0% → 100%

6. ✅ Загрузка завершена
   └─ Проверить: File integrity (SHA512) ✅

7. ✅ Предложен перезапуск
   └─ Проверить: Кнопка "Перезапустить и установить"

8. ✅ Установка произошла
   └─ Проверить: app.getVersion() = "1.0.1"

9. ✅ Приложение запустилось без ошибок
   └─ Проверить: No exceptions in console
```

## 🛡️ Фаза 4: Проверка отказоустойчивости

### Тест-кейсы ошибок

| Сценарий | Проверка | Ожидаемое поведение |
|----------|----------|-------------------|
| **Нет интернета** | Отключить WiFi | Ошибка логируется, приложение работает |
| **Сервер недоступен** | curl на неправильный URL | Ошибка, повтор через час |
| **Повреждён latest.yml** | YAML синтаксис неверный | Ошибка парсинга, приложение работает |
| **Отсутствует exe** | Удалить файл с сервера | 404 ошибка, приложение работает |
| **Неверный SHA512** | Изменить хеш в latest.yml | Ошибка проверки целостности |

## 📚 Фаза 6: Production документация

### Что нужно документировать

1. **URL сервера обновлений**
   ```
   https://updates.mvs.app/releases
   ```

2. **Структура файлов на сервере**
   ```
   latest.yml
   MVSSetup-X.X.X.exe
   MVSSetup-X.X.X.exe.blockmap
   ```

3. **Пример latest.yml для 1.0.1**
   ```yaml
   version: 1.0.1
   files:
     - url: MVSSetup-1.0.1.exe
       sha512: '...'
       size: ...
   path: MVSSetup-1.0.1.exe
   sha512: '...'
   releaseDate: '2026-07-10T10:00:00.000Z'
   ```

4. **Инструкции по выпуску**
   - Обновить version в package.json
   - Запустить npm run build
   - Вычислить SHA512
   - Обновить latest.yml
   - Загрузить файлы на сервер
