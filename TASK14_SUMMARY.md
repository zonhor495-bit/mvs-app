## Где это находится в приложении

### Интерфейс пользователя:
1. **Меню:** `⚙️ Settings → Backup Settings`
   - Включить/отключить шифрование
   - Установить пароль для резервных копий
   - Выбрать частоту копирования (ежедневно/еженедельно/при закрытии)
   - Установить максимальное количество копий

2. **Основной экран:** `BackupManager` (новая вкладка)
   - Кнопка "Создать резервную копию" - делает копию прямо сейчас
   - Список всех резервных копий с датами
   - Кнопка "⤺" (восстановить) для каждой копии
   - Кнопка "🗑" (удалить) для каждой копии

3. **При восстановлении:** Пошаговый мастер
   - Выбор файла
   - Ввод пароля (если зашифровано)
   - Выбор разделов для восстановления
   - Проверка целостности
   - Отслеживание прогресса

## Как использовать

### Создание резервной копии:
```
1. BackupManager → Кнопка "Создать резервную копию"
2. Выбрать опции (шифрование, пароль)
3. Нажать "Создать"
4. Дождаться завершения
5. Файл сохранится как backup_ДАТА_ВРЕМЯ.zip
```

### Восстановление из резервной копии:
```
1. BackupManager → Кнопка "⤺" (восстановить) рядом с нужной копией
2. ИЛИ кнопка "Восстановить" → выбрать файл вручную
3. Если зашифровано → ввести пароль и нажать "Разблокировать"
4. Выбрать какие разделы восстановить (опционально)
5. Нажать "Проверить" → убедиться что ошибок нет
6. Нажать "Восстановить"
7. Дождаться завершения (прогресс-бар показывает %)
8. Приложение автоматически перезагрузится с восстановленными данными
```

### Выборочное восстановление (только нужные разделы):
```
1. На шаге "Выбор разделов" снять галочки со всех кроме нужных
2. Пример: если нужны только клиенты, оставить отмеченным только "Clients"
3. Остальные разделы (Washers, Orders и т.д.) не изменятся
```

### Использование пароля:
```
1. Settings → Backup Settings → включить "Enable Password Protection"
2. Установить пароль (минимум 8 символов, буквы+цифры рекомендуется)
3. Все новые резервные копии будут зашифрованы этим паролем
4. При восстановлении будет запрос на ввод пароля
```

## Рекомендации по использованию

### 🎯 Best Practices:

1. **Создавайте резервные копии регулярно**
   - Включите автоматическое копирование в Settings
   - Выберите "Daily" (ежедневное) или "Weekly" (еженедельное)

2. **Используйте пароль для защиты**
   - Включите "Password Protection" в Settings
   - Запомните пароль (его нельзя восстановить!)
   - Используйте сильный пароль (8+ символов)

3. **Проверяйте резервные копии**
   - Периодически пытайтесь восстановить (даже если не нужно)
   - Убедитесь что можете восстановиться за 5-10 минут

4. **Храните копии в разных местах**
   - Локально на компьютере (автоматически)
   - На USB-флешке (скопировать файл)
   - В облаке (Google Drive, OneDrive и т.д.)

5. **Тестируйте на другом ПК**
   - Возьмите резервную копию на чужой компьютер
   - Убедитесь что там всё восстанавливается корректно

## Часто задаваемые вопросы

### Q: Что произойдёт если я забуду пароль?
**A:** К сожалению, зашифрованную копию нельзя будет восстановить. Пароль хранится только в вашей памяти. Это нормально для безопасности. Храните пароль в безопасном месте.

### Q: Как большой может быть резервная копия?
**A:** Обычно 1/10 от размера исходных данных благодаря сжатию ZIP. Например 100 МБ данных = ~10 МБ резервной копии.

### Q: Можно ли восстановить только клиентов?
**A:** Да! На шаге "Выбор разделов" выберите только нужные разделы. Остальные данные не изменятся.

### Q: Сколько памяти занимает каждая копия?
**A:** По умолчанию хранится максимум 10 копий. Можно изменить в Settings (maxBackupCount).

### Q: Работает ли на Windows/macOS/Linux?
**A:** Да, система полностью кроссплатформенная. Резервные копии совместимы между всеми ОС.

### Q: Можно ли загрузить резервную копию в облако?
**A:** Да! Скачайте файл backup_*.zip и загрузите в Google Drive, OneDrive, Dropbox и т.д. Позже можно скачать и восстановить.

### Q: Что происходит при восстановлении?
**A:** 
1. Всё существующие данные заменяются данными из резервной копии
2. Перед восстановлением можно создать снимок текущего состояния
3. Приложение перезагружается после завершения
