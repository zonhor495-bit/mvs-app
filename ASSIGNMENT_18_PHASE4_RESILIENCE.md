# Assignment 18 Phase 4: Тестирование отказоустойчивости

## 🛡️ Цель Phase 4

Убедиться что система обновлений **не крашится** и продолжает работать при:
- Отсутствии интернета
- Недоступности сервера
- Повреждённых файлах
- Ошибках загрузки
- Других критических ошибках

### Сценарий 3: Повреждённый latest.yml

**Тест:** Проверка поведения при неверном YAML синтаксисе

#### Подготовка

```
1. На сервере обновлений повредить latest.yml
   Например, заменить ':' на '?':
   
   ❌ НЕВЕРНО:
   version? 1.0.1
   files?
     - url? https://...

   version: 1.0.1
   files:
     - url: https://...
```

### Сценарий 5: Неверный SHA512 хеш

**Тест:** Проверка поведения при несовпадении хешей

```

   sha512: 'AAAABBBBCCCCDDDDEEEE...' (не совпадает с файлом)
   
   ✅ ВЕРНО:
   sha512: '...' (совпадает с файлом)
```

```
│                                                  │
│ Проверка:                                       │
│ ✓ Файл загружается                              │
│ ✓ При проверке целостности — ОШИБКА            │
│ ✓ Приложение НЕ крашится                       │
│ ✓ Диалог закрывается или показывает ошибку     │
```
