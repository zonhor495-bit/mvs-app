### 2. RELEASE_CHECKLIST.md

**Для:** Release Manager'ов  
**Содержит:** Пошаговый checklist выпуска новой версии

```markdown
# Release Checklist

## Перед релизом

- [ ] Все тесты пройдены (unit, integration, e2e)
- [ ] Нет критических bugs
- [ ] Документация обновлена
- [ ] Логи changelog подготовлены

## Выпуск версии X.Y.Z

1. **Обновить версию**
   ```bash
   # Измените в package.json:
   "version": "X.Y.Z"
   ```

2. **Собрать приложение**
   ```bash
   npm run build
   npm run dist:win
   ```

3. **Вычислить SHA512**
   ```bash
   # PowerShell:
   (Get-FileHash -Path "build/MVSSetup-X.Y.Z.exe" -Algorithm SHA512).Hash
   ```

4. **Создать Release на GitHub**
   ```bash
   gh release create vX.Y.Z \
     build/MVSSetup-X.Y.Z.exe \
     build/MVSSetup-X.Y.Z.exe.blockmap \
     build/latest.yml \
     --title "MVS X.Y.Z" \
     --notes "Release notes here" \
     --latest
   ```

5. **Проверить что файлы доступны**
   ```bash
   curl -I https://github.com/USERNAME/mvs-app/releases/download/vX.Y.Z/MVSSetup-X.Y.Z.exe
   # Должен быть 200 OK
   ```

6. **Информировать пользователей** (опционально)
   - Email уведомление
   - В-приложение уведомление
   - Социальные сети

## После релиза

- [ ] Проверить что обновление обнаруживается
- [ ] Проверить что диалог показывается
- [ ] Проверить что загрузка работает
- [ ] Проверить что установка прошла успешно
- [ ] Документировать issues (если есть)
- [ ] Подготовить патч (если нужен hotfix)

## Откат (если критическая ошибка)

- [ ] Создать новый release с предыдущей версией
- [ ] Обновить latest.yml
- [ ] Информировать пользователей
- [ ] Post-mortem анализ
```

### 3. USER_GUIDE_UPDATES.md

**Для:** End Users  
**Содержит:** Что ожидать при обновлении

```markdown
# Руководство пользователя: Автоматические обновления

## Что произойдёт?

Когда вы запустите MVS, приложение автоматически проверит наличие обновлений.

Если обновление доступно, вы увидите красивый диалог:

```
┌─────────────────────────────┐
│ 🎉 Обновление доступно    │
│                             │
│ Версия 1.0.0 → 1.0.1      │
│                             │
│ [Обновить сейчас]           │
│ [Напомнить позже]           │
└─────────────────────────────┘
```

## Опция 1: Обновить сейчас

1. Нажмите "Обновить сейчас"
2. Подождите загрузки (видна прогресс-бар)
3. Нажмите "Перезапустить и установить"
4. Приложение установится автоматически
5. Запустится новая версия

Это займёт 5-15 минут в зависимости от скорости интернета.

## Опция 2: Напомнить позже

1. Нажмите "Напомнить позже"
2. Диалог закроется
3. Приложение продолжит работать
4. Проверка повторится через час или при перезапуске

## Важное

- ✅ Ваши данные **не потеряются**
- ✅ Ярлыки **останутся на месте**
- ✅ Настройки **сохранятся**
- ✅ **Не требуется** ничего делать вручную

## Если что-то пошло не так

1. Приложение **не крашится** при ошибке
2. Если загрузка прервалась:
   - Приложение продолжит работать с текущей версией
   - Проверка повторится через час

3. Вы всегда можете **скачать установщик вручную** на:
   https://github.com/USERNAME/mvs-app/releases

## Как проверить версию?

В приложении откройте:
- Menu → Help → About

Или в Developer Console:
- F12 → Console → electron.versions
```

### 4. SUPPORT_TROUBLESHOOTING.md

**Для:** Support Team  
**Содержит:** Решение типичных проблем

```markdown
# Troubleshooting: Проблемы с обновлениями

## Проблема: Диалог не появляется

### Причины и решения

1. **Версия уже актуальна**
   - Проверка: Меню → Help → About
   - Если версия = последняя, диалог не нужен

2. **Нет интернета**
   - Проверка: Есть ли доступ в интернет?
   - Решение: Подключитесь в интернет и перезапустите

3. **Сервер обновлений недоступен**
   - Проверка: curl https://github.com/.../releases/latest.yml
   - Если 404 → файл не найден на сервере
   - Решение: Проверить сервер, переупустить release

4. **Firewall блокирует обновления**
   - Проверка: Есть ли правила firewall?
   - Решение: Добавить исключение для MVS.exe

---

## Проблема: Загрузка зависает или очень медленная

### Причины и решения

1. **Медленный интернет**
   - Решение: Ждать (обновление в фоне, приложение работает)
   - Или нажать "Позже" и попробовать позже

2. **Большой файл обновления**
   - Обычно: 100-150 MB
   - При медленном интернете может быть медленно
   - Решение: Ждать или попробовать позже

3. **Прокси или VPN**
   - Может медлить при использовании прокси
   - Решение: Попробовать без прокси/VPN

---

## Проблема: Ошибка SHA512 / Integrity check failed

### Решение

1. Удалить загруженный файл
   ```
   %APPDATA%/MVS/updates/
   ```

2. Попробовать обновление ещё раз
   - Обычно проходит со второй попытки

3. Если ошибка повторяется:
   - Написать в поддержку
   - Указать версию ОС и приложения
   - Приложить логи (startup.log)

---

## Проблема: Приложение крашится при обновлении

### Решение

1. Переустановить вручную:
   ```
   1. Скачать MVSSetup-X.X.X.exe с GitHub
   2. Запустить установщик
   3. Следовать инструкциям
   ```

2. Если проблема не решена:
   - Написать в поддержку с:
   - startup.log
   - Скриншот ошибки
   - Версия Windows

---

## Проблема: Данные потеряны после обновления

### Решение

1. **Важно:** Данные хранятся в %APPDATA%/MVS/
   - При обновлении они НЕ удаляются
   - Если потеряны → скорее всего другая причина

2. Проверить наличие файлов:
   ```
   %APPDATA%/MVS/database.*
   ```

3. Восстановить из backup (если есть):
   ```
   Меню → Backup & Restore → Restore from file
   ```

4. Если backup нет:
   - Контактировать поддержку
   - Может быть возможно восстановление

---

## Кроссверкл логов (для диагностики)

### Где найти логи

```
Windows: %APPDATA%/MVS/startup.log
macOS: ~/Library/Application Support/MVS/startup.log
Linux: ~/.config/MVS/startup.log
```

### Что искать в логах при проблемах

```
ERROR при обновлении:
[updater] Error: ... ← информация об ошибке

Сеть:
[updater] Check for updates...
[updater] Error: Network timeout ← нет интернета

Файл:
[updater] Error: 404 Not Found ← файл не найден на сервере
[updater] Error: SHA512 mismatch ← повреждён файл

База:
[app] Error loading database ← проблема с БД
```

---

## Когда контактировать поддержку

Необходимо:
- Скопировать text из dialoga error
- Приложить startup.log
- Указать версию Windows
- Указать версию MVS
- Описать что делали перед ошибкой
```

### 5. METRICS_AND_MONITORING.md

**Для:** DevOps / Аналитика  
**Содержит:** Метрики и мониторинг обновлений

```markdown
# Метрики и мониторинг обновлений

## Ключевые метрики

1. **Update Detection Rate**
   - Сколько % пользователей обнаружили обновление?
   - Цель: >95%

2. **Update Adoption Rate**
   - Сколько % пользователей установили обновление?
   - Цель: >80% за 7 дней

3. **Update Failure Rate**
   - Сколько % попыток обновления завершились с ошибкой?
   - Цель: <1%

4. **Average Update Time**
   - Сколько времени занимает обновление?
   - Обычно: 5-15 минут

5. **Rollback Rate**
   - Сколько пользователей откатились на старую версию?
   - Цель: <0.1%

## Где собирать метрики

### GitHub Releases API

```bash
# Скачивания файлов:
curl https://api.github.com/repos/USERNAME/mvs-app/releases/latest | jq '.assets'

# Все релизы:
curl https://api.github.com/repos/USERNAME/mvs-app/releases
```

### Приложение (встроенное телеметрия)

В startup.log логируется:
```
[telemetry] Update available
[telemetry] Update started
[telemetry] Update completed
[telemetry] Update failed
```

### Сервер обновлений

Если используется custom server:
- Мониторить логи сервера
- Отслеживать 404 ошибки (отсутствующие версии)
- Отслеживать 5xx ошибки (проблемы сервера)

## Алерты

Настроить алерты для:
- Update failure rate > 5%
- Server down (503 > 10%)
- Unusual download patterns
```
