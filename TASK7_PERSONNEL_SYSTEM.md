##### **WorkerProfileCard.tsx** — Модальное окно профиля

```
Табы:

  4. 💵 Зарплата — история выплат, итоги за период
```

##### **AttendanceTable.tsx** — Таблица присутствия
```
Столбцы:
  - Имя/статус
  - Текущий бокс
  - Машин сегодня
  - Заработок сегодня
  - Время начала работы
  - Действия (открыть профиль)
```

##### **Washers.tsx** — Основной модуль (5 табов)
```
1. 👷 Сотрудники (Overview)
   - AttendanceTable (полная таблица присутствия)
   - Правая колонка: уведомления (новые события)
   - Карточки сотрудников: ставка, машины, заработок, статус
   - Действия: редактировать, удалить, открыть профиль
   - Для менеджера: смена временного бокса, если работает

2. 📅 Смены (Shifts)
   - Кнопки: "Открыть день", "Закрыть день"
   - Статистика: работают, отсутствуют, выходной
   - Таблица: сотрудник, статус, премия, штраф, итог
   - Действия: редактировать смену, удалить

3. 🕘 История (History)
   - Фильтр по сотруднику
   - Фильтр по месяцу
   - Таблица: дата, действие, детали, результат
   - Полный журнал всех операций

4. 💵 Зарплата (Salary)
   - Выбор месяца
   - Форма: сотрудник, ставка, дни, бонусы, штрафы
   - Итог к выплате
   - Кнопка "Выплатить" → логирование в ActionLog
   - История выплат

5. 📈 Аналитика (Analytics)
   - Средняя ставка, всего сотрудников
   - Топ по машинам, по доходу
   - График заработков (день/неделя/месяц)
   - Эффективность работы
```

### 🎯 Рабочие сценарии (Use Cases)

#### **Сценарий 1: Добавление нового сотрудника**
```
1. Открить Tab "Сотрудники"
2. Нажать "+ Добавить"
3. Заполнить:
   - Имя: "Иван"
   - Телефон: "+7 (123) 456-78-90"
   - Ставка за день: 500
   - Основной бокс: "Бокс 1"
   - Схема оплаты: "Процент" / "50%"
4. Нажать "Сохранить"
✓ Сотрудник добавлен в список, отобразится карточка
✓ ActionLog: добавлена запись "add_worker"
✓ Уведомление: "Добавлен новый сотрудник Иван"
```

#### **Сценарий 2: Открыть день (начало смены)**
```
1. Перейти в Tab "Смены"
2. Нажать "📅 Открыть день"
3. ShiftStartModal: выбрать статус для каждого сотрудника
   - Работает (по умолчанию)
   - Отсутствует
   - Выходной
   - Больничный
4. Нажать "Готово"
✓ Смены созданы, отображаются в таблице
✓ Для каждого "работает" → можно менять бокс
✓ ActionLog: "shift_start"
```

#### **Сценарий 3: Смена временного бокса во время работы**
```
1. Tab "Сотрудники" → если сотрудник работает
2. Карточка сотрудника → Dropdown "Временный бокс"
3. Выбрать новый бокс
✓ Обновляется currentStatus.currentBoxId
✓ ActionLog: "box_change" с деталями
✓ Уведомление: "Иван переведён в Бокс 2"
```

#### **Сценарий 4: Открыть профиль сотрудника (детальный просмотр)**
```
1. Tab "Сотрудники" → кнопка "Открыть профиль" на карточке
   или Tab "История" → клик по сотруднику
2. WorkerProfileCard открывается модально
3. Выбрать таб:
   - 👤 Профиль: все данные, схема оплаты
   - 📅 Смены: список смен за месяц
   - 🚗 Заказы: выполненные заказы, доход по каждому
   - 💵 Зарплата: история выплат
   - 📋 Логи: все действия (создание → изменения → выплаты)
✓ Полный профиль работника с историей
```

#### **Сценарий 5: Выплата зарплаты**
```
1. Tab "Зарплата"
2. Выбрать месяц (по умолчанию текущий)
3. Форма для сотрудника:
   - Ставка: 500 (автозаполнена)
   - Рабочих дней: 20
   - Премии: 1000 (опционально)
   - Штрафы: 0 (опционально)
   - Итог: 500*20 + 1000 - 0 = 11000
4. Нажать "Выплатить"
✓ Выплата зарегистрирована
✓ ActionLog: "salary_payout" с суммой и датой
✓ Уведомление: "Выплачена зарплата Иван: 11000 руб"
✓ История выплат обновляется
```

#### **Сценарий 6: Просмотр истории действий**
```
1. Tab "История"
2. Фильтры (опционально):
   - По сотруднику
   - По месяцу
3. Таблица показывает:
   - 12.01.2025 | Иван | shift_start | начало смены
   - 12.01.2025 | Иван | box_change | Бокс 1 → Бокс 2
   - 12.01.2025 | Иван | order_link | заказ СУ-2025-001
   - 20.01.2025 | Иван | salary_payout | 11000 руб
✓ Полный аудит всех операций
```

### 🔌 Интеграция со Store (localStorage)

**Keys используются:**
```javascript
KEYS.washers        // [{ id, name, phone, dailyRate, paymentSettings, ... }]
KEYS.shifts         // [{ id, washerId, date, status, bonus, penalty, ... }]
KEYS.actionLogs     // [{ id, washerId, action, details, createdAt, ... }]
KEYS.notifications  // [{ id, washerId, message, severity, read, ... }]
KEYS.washerCurrentStatus  // { washerId: { status, currentBoxId, ... } }
```

### ✨ Особенности реализации

1. **Reactive updates**: все изменения сразу отображаются благодаря `version` state
2. **Persistent storage**: все данные сохраняются в localStorage
3. **Real-time notifications**: уведомления добавляются при каждом действии
4. **Flexible payment schemes**: поддержка процента, зарплаты, смешанной и фиксированной системы
5. **Full audit trail**: каждое действие логируется с timestamp
6. **Role-based access**: менеджер vs администратор vs другие роли
7. **Performance optimized**: использование useMemo для больших списков
